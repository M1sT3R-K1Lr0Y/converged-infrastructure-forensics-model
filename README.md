# Converged Infrastructure Forensics Model (CIFM)

The Converged Infrastructure Forensics Model (CIFM) is an investigative framework for reconstructing adversary activity across converged infrastructure ecosystems — cloud, telecom, edge, IoT/IIoT, and OT/ICS — using identity-centric analysis, distributed telemetry, and human-validated reasoning.

## About the Framework

CIFM addresses a critical gap in digital forensics methodology: existing models were designed for host-centric environments and do not provide a unified approach for reconstructing adversary activity across interconnected infrastructure domains.

CIFM introduces four primary structural contributions:
- **Unified Evidence Manifest (UEM)** — normalized, domain-agnostic evidence schema
- **HITL Validation Tiers** — constrained AI assistance with mandatory human verification
- **Iterative, non-linear workflow** — explicitly models real investigation complexity
- **Visibility & Trust Boundary Matrix** — pre-investigation telemetry assessment tool
